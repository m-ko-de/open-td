# Contributing to Open TD

Thanks for your interest in contributing! Please follow these guidelines to make collaboration smooth.

1. Fork the repository and create a feature branch from `main`.

2. Run tests locally and ensure they pass before submitting PRs:

```bash
pnpm install
pnpm run test:run
```

3. Code style
- Use TypeScript and keep consistent style with existing files.
- Respect existing module boundaries and naming conventions.

4. Commit messages
- Use descriptive commit messages. Consider the pattern: `feat:`, `fix:`, `docs:`, `chore:`.

5. Pull requests
- Open a PR against `main` with a clear description and screenshots if applicable.
- Link related issues and describe testing steps.
- CI will run tests and build; please address any failing checks.

6. Issues
- Search existing issues before filing a new one.
- Provide steps to reproduce and relevant logs.

7. Code of Conduct
- Be respectful and constructive. If you need to report a problem, open an issue or contact the maintainers privately.
